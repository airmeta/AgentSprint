# AgentSprint Docker 部署教程与步骤

本文档记录将 AgentSprint 的 `admin`、`api`、`mcp` 三个服务部署到 101 服务器的流程，适用于当前 Docker Compose 部署方式。

> 当前正式部署目标为 `192.168.80.101`。`192.168.100.161` 不作为当前部署目标；如后续重新启用，需要先确认远端 `.env`、Docker 版本、Compose 行为和数据库连通性。

## 1. 部署目标

- 服务器：`192.168.80.101`
- SSH 用户：`root`
- 远端部署根目录：`/opt/agentsprint-deploy`
- 远端 Compose 目录：`/opt/agentsprint-deploy/docker`
- 部署包路径：`/opt/agentsprint-deploy/agentsprint-docker-deploy.tgz`
- 前端服务：`agentsprint-admin`，宿主端口 `5999`，容器端口 `80`
- API 服务：`agentsprint-api`，宿主端口 `5000`，容器端口 `5000`
- MCP 服务：`agentsprint-mcp`，宿主端口 `5010`，容器端口 `5010`

## 2. 本地前置检查

在仓库根目录执行：

```powershell
cd F:\AI\AgentSprint
git status --short
```

确认当前改动符合预期后，执行后端和前端验证：

```powershell
dotnet test .\src\api\AgentSprint.Tests\AgentSprint.Tests.csproj --no-restore

cd .\src\admin
corepack pnpm -F @vben/web-tdesign run typecheck
cd ..\..
```

如修改了前端页面或构建相关内容，还需要执行生产构建：

```powershell
cd .\src\admin
corepack pnpm -F @vben/web-tdesign run build
cd ..\..
```

注意：前端 `package.json` 和 `pnpm-workspace.yaml` 位于 `F:\AI\AgentSprint\src\admin`，不要在仓库根目录直接执行 `corepack pnpm -F ...`，否则会报 `ERR_PNPM_NO_PKG_MANIFEST`。

Codex 执行时建议预先允许以下命令提权：

- `corepack pnpm ...`：Corepack 会访问 `C:\Users\23129\AppData\Local\node\corepack\lastKnownGood.json`。
- `scp -i C:\Users\23129\.ssh\id_ed25519 ...`：需要读取 SSH key 并连接远端 22 端口。
- `ssh -i C:\Users\23129\.ssh\id_ed25519 ...`：需要读取 SSH key 并连接远端 22 端口。
- `Remove-Item -Recurse -Force .\deploy\docker\artifacts ...`：会清理本地旧部署产物目录，执行前需确认只删除 `deploy\docker\artifacts`。

## 3. 生成部署产物

清理旧 artifact：

```powershell
Remove-Item -Recurse -Force .\deploy\docker\artifacts -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Force .\deploy\docker\artifacts\api | Out-Null
New-Item -ItemType Directory -Force .\deploy\docker\artifacts\mcp | Out-Null
New-Item -ItemType Directory -Force .\deploy\docker\artifacts\admin | Out-Null
```

发布 API：

```powershell
dotnet publish .\src\api\AgentSprint.Entry\AgentSprint.Entry.csproj `
  -c Release `
  -o .\deploy\docker\artifacts\api `
  --no-restore
```

发布 MCP：

```powershell
dotnet publish .\src\api\AgentSprint.Mcp\AgentSprint.Mcp.csproj `
  -c Release `
  -o .\deploy\docker\artifacts\mcp `
  --no-restore
```

构建并复制前端：

```powershell
cd .\src\admin
corepack pnpm -F @vben/web-tdesign run build
cd ..\..

Copy-Item -Recurse -Force .\src\admin\apps\web-tdesign\dist\* .\deploy\docker\artifacts\admin\
```

可检查三个 artifact 的文件数和大小：

```powershell
Get-ChildItem .\deploy\docker\artifacts -Directory | ForEach-Object {
  $files = Get-ChildItem $_.FullName -Recurse -File
  [PSCustomObject]@{
    Name = $_.Name
    Files = $files.Count
    Bytes = ($files | Measure-Object Length -Sum).Sum
  }
} | Format-Table -AutoSize
```

## 4. 打包部署目录

部署包只需要包含 `deploy/docker` 目录中的 Compose 文件、Dockerfile、nginx 配置和三个 artifact。

```powershell
if (Test-Path .\agentsprint-docker-deploy.tgz) {
  Move-Item -Force .\agentsprint-docker-deploy.tgz .\agentsprint-docker-deploy.tgz.prev
}

tar -czf .\agentsprint-docker-deploy.tgz -C .\deploy docker
Get-Item .\agentsprint-docker-deploy.tgz | Select-Object FullName,Length,LastWriteTime
```

可抽查压缩包内容：

```powershell
tar -tzf .\agentsprint-docker-deploy.tgz | Select-Object -First 30
```

在 Windows 上，`tar -tzf ... | Select-Object -First ...` 可能因为管道提前关闭返回非 0。只要包元数据和预期条目已经打印出来，此处可视为抽查通过。

## 6. 远端部署

优先使用发布包内的远端部署脚本。脚本会自动备份当前 `docker` 目录、清理解包、保留远端 `.env`、执行 `docker compose build --no-cache`、`docker compose up -d`，并做基础健康检查。

```powershell
ssh -i C:\Users\23129\.ssh\id_ed25519 root@192.168.80.101 'bash /opt/agentsprint-deploy/deploy.sh'
```

注意：PowerShell 中远端命令必须用单引号包住，否则 `$(date ...)` 会被本地 PowerShell 提前解析。

如果远端还没有 `/opt/agentsprint-deploy/deploy.sh`，先从发布包中解出脚本：

```powershell
ssh -i C:\Users\23129\.ssh\id_ed25519 root@192.168.80.101 'set -e; cd /opt/agentsprint-deploy; tar -xzf agentsprint-docker-deploy.tgz docker/deploy.sh; cp docker/deploy.sh deploy.sh; chmod 755 deploy.sh docker/deploy.sh; bash /opt/agentsprint-deploy/deploy.sh'
```

当前 Compose 会构建以下镜像：

- `agentsprint-admin:latest`
- `agentsprint-api:latest`
- `agentsprint-mcp:latest`

远端必须存在 `/opt/agentsprint-deploy/docker/.env`，并包含 `AGENTSPRINT_DB_PASSWORD=`。脚本会在清理解包前备份并恢复旧 `.env`；全新服务器没有旧 `.env` 时，需要先人工准备该文件，不要在终端输出敏感值。

## 7. 部署验证

检查容器状态：

```powershell
ssh 192.168.80.101 'docker ps --filter name=agentsprint --format "table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"'
```

检查镜像 ID 和大小：

```powershell
ssh 192.168.80.101 'docker images --format "table {{.Repository}}\t{{.ID}}\t{{.Size}}" | grep agentsprint'
```

检查 HTTP 响应：

```powershell
ssh 192.168.80.101 'curl -I --max-time 10 http://127.0.0.1:5999'
ssh 192.168.80.101 'curl -i --max-time 10 http://127.0.0.1:5000'
ssh 192.168.80.101 'curl -i --max-time 10 http://127.0.0.1:5010/mcp'
```

期望结果：

- `http://127.0.0.1:5999` 返回 `200 OK`
- `http://127.0.0.1:5000` 根路径可能返回 `404 Not Found`，只要容器运行且 Kestrel 正常响应即可
- `http://127.0.0.1:5010/mcp` 返回 MCP 服务 JSON 信息

检查日志：

```powershell
ssh 192.168.80.101 'docker logs --tail 80 agentsprint-api'
ssh 192.168.80.101 'docker logs --tail 80 agentsprint-mcp'
ssh 192.168.80.101 'docker logs --tail 80 agentsprint-admin'
```

## 8. 回滚方式

如果新版本启动失败，使用最近一次备份目录回滚。先查看备份：

```powershell
ssh 192.168.80.101 'ls -dt /opt/agentsprint-deploy/docker.bak.* | head'
```

假设要回滚到 `/opt/agentsprint-deploy/docker.bak.20260608105421`：

```powershell
ssh 192.168.80.101 'set -e; cd /opt/agentsprint-deploy; mv docker docker.failed.$(date +%Y%m%d%H%M%S); cp -a docker.bak.20260608105421 docker; cd docker; docker compose build --no-cache; docker compose up -d'
```

回滚后重新执行第 7 节验证。
