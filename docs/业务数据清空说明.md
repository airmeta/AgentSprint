# 业务数据清空说明

本文记录 AgentSprint 在保留系统管理数据的前提下，清空项目、需求、端、评审、任务大厅、测试和测试计划相关业务数据的 SQL。

## 适用场景

- 需要保留用户、角色、菜单、权限、字典、运行环境、提示词模板等系统管理数据。
- 需要重新录入项目、端、模块、需求、评审、任务、测试计划、测试执行和缺陷数据。
- 不需要整库重建，也不需要删除 `sys_*` 系统管理表。

## 执行前备份

清空数据前必须先备份当前 `agentsprint` 数据库。

```bash
mysqldump -uroot -p --databases agentsprint > agentsprint-backup.sql
```

Docker MySQL 示例：

```bash
docker exec mysql_server sh -c 'MYSQL_PWD=123456 mysqldump -uroot --databases agentsprint' > agentsprint-backup.sql
```

## 清空 SQL

按下面顺序执行。该脚本只清空敏捷研发和测试业务表，不清空系统管理表。

```sql
SET FOREIGN_KEY_CHECKS = 0;

-- 测试 / 测试计划 / 缺陷
TRUNCATE TABLE test_execution;
TRUNCATE TABLE test_plan;
TRUNCATE TABLE sprint_bug;

-- 任务大厅
TRUNCATE TABLE sprint_task_lease;
TRUNCATE TABLE sprint_development_task;

-- 需求 / 评审 / 反馈 / 优化建议
TRUNCATE TABLE sprint_requirement_review;
TRUNCATE TABLE sprint_requirement_feedback;
TRUNCATE TABLE sprint_feature_suggestion;
TRUNCATE TABLE sprint_requirement;

-- 端 / 模块
TRUNCATE TABLE sprint_feature_module;
TRUNCATE TABLE sprint_project_endpoint;

-- 项目成员 / 项目
TRUNCATE TABLE sprint_project_member;
TRUNCATE TABLE sprint_project;

SET FOREIGN_KEY_CHECKS = 1;
```

## 可选清空 Skill

如果 Skill 配置也需要重新维护，再额外执行：

```sql
TRUNCATE TABLE sprint_skill;
```

## 不应清空的表

通常不要清空这些表：

- `sys_user`
- `sys_role`
- `sys_menu`
- `sys_permission`
- `sys_user_role`
- `sys_role_menu`
- `sys_role_permission`
- `sys_configuration`
- `sys_dictionary_type`
- `sys_dictionary_item`
- `sys_runtime_environment`
- `sys_runtime_environment_container`
- `sys_prompt_template`
- `assignment`

这些表属于系统管理和运行配置。清空后会影响登录、菜单、权限、字典、部署环境和 MCP 提示词。

## 验证语句

执行后可用下面 SQL 确认业务数据已清空：

```sql
SELECT COUNT(*) AS project_count FROM sprint_project;
SELECT COUNT(*) AS endpoint_count FROM sprint_project_endpoint;
SELECT COUNT(*) AS module_count FROM sprint_feature_module;
SELECT COUNT(*) AS requirement_count FROM sprint_requirement;
SELECT COUNT(*) AS review_count FROM sprint_requirement_review;
SELECT COUNT(*) AS task_count FROM sprint_development_task;
SELECT COUNT(*) AS test_plan_count FROM test_plan;
SELECT COUNT(*) AS test_execution_count FROM test_execution;
SELECT COUNT(*) AS bug_count FROM sprint_bug;
```

系统管理数据应仍然存在：

```sql
SELECT COUNT(*) AS user_count FROM sys_user;
SELECT COUNT(*) AS menu_count FROM sys_menu;
SELECT COUNT(*) AS role_count FROM sys_role;
```
