# Git管理说明

Git管理用于把项目配置里的仓库地址和Git账户从手工输入改为统一数据源选择，并提供基础分支操作能力。

## 功能入口

- Git账户管理：维护访问仓库使用的账户编码、账户名称、Git用户名、访问令牌和状态。
- Git仓库管理：维护仓库编码、仓库名称、仓库地址、默认分支、绑定账户和本地工作副本路径。
- 项目管理：新增或编辑项目时选择Git仓库和Git账户，系统会把仓库地址、用户名和令牌同步为项目快照，兼容已有任务提示词和Worker运行逻辑。

## 仓库分支操作

Git仓库管理提供三个基础操作：

- 新增分支：从默认分支或指定来源分支创建新分支，并推送到远端。
- 删除分支：先创建并推送备份分支，再删除远端和本地目标分支；未填写备份分支时自动生成 `backup/分支名-时间`。
- 读取推送记录：读取指定分支或全部远端分支的最近提交记录，并保存读取快照。

## 配置建议

- Git账户的访问令牌建议使用最小权限令牌，仅开放目标仓库的读写分支权限。
- Git仓库的本地路径为空时，服务会使用临时目录克隆工作副本；生产环境建议配置固定路径，便于复用和排查。
- 删除分支前请确认备份分支命名规范，避免和已有远端分支冲突。

## 验证

完成配置后，可在项目管理中新建项目，选择已启用的Git仓库与Git账户；保存后项目详情会展示所选Git仓库、Git账户和同步后的仓库地址。
