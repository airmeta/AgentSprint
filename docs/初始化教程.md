# AgentSprint 系统初始化教程

本文档用于指导研发人员完成 Codex、项目仓库、AgentSprint 协同平台和 MCP 自动化推进链路的初始化。

## 前置条件

- 已获得 Codex 客户端安装包或官方下载入口。
- 已获得 AgentSprint 协同平台账号。
- 已获得目标项目的 Git 仓库地址和仓库访问权限。
- 本机已安装 Git，并可以访问项目仓库。

## 初始化流程

### 1. 安装并登录 Codex

在本机安装 Codex 客户端，启动后使用个人账号登录。登录完成后，确认 Codex 可以正常创建会话并访问本机文件系统。

### 2. 获取项目仓库地址并拉取代码

从项目负责人或协同平台获取当前项目所在的 Git 仓库地址，在本机选择一个稳定的工作目录后拉取仓库代码。

```powershell
git clone <项目仓库地址>
cd <项目目录>
```

如果仓库已经存在，进入项目目录后更新代码：

```powershell
git pull
```

### 3. 在 Codex 中添加项目

进入 Codex 对话界面，选择添加新项目，并选择“使用现有文件夹”。在文件夹选择窗口中，选中上一步拉取的项目根目录。

项目添加完成后，后续所有任务推进提示词都应粘贴到基于该项目创建的聊天对话中。

### 4. 设置完全访问权限

为该项目会话设置完全访问权限。完全访问权限是自动化推进的前提，Codex 需要在任务执行过程中读取项目文件、修改代码、运行验证命令，并在需要时写入 MCP 配置。

如果没有开启完全访问权限，Codex 可能无法完成以下操作：

- 写入或更新 MCP 配置。
- 自动修改项目文件。
- 执行本地构建、测试或验证命令。
- 持续接取并推进后续任务。

### 5. 登录协同平台并执行 MCP 接入提示词

登录 AgentSprint 协同平台，进入 MCP 接入或任务推进相关页面，复制平台生成的 MCP 接入提示词。

回到 Codex 中基于当前项目的聊天对话，确认会话已经具备完全访问权限，然后粘贴 MCP 接入提示词并回车执行。Codex 会根据提示词完成 MCP 接入配置。

MCP 接入成功后，Codex 才能调用协同平台工具读取任务详情、注册会话、回写任务状态和继续领取后续工作。

### 6. 在任务大厅接取任务

进入 AgentSprint 协同平台的任务大厅，选择当前需要处理的任务并接取。接取后，任务会进入当前账号的任务列表。

### 7. 在我的列表中选择任务并点击任务推进

进入“我的任务”或“我的列表”，找到已接取的任务，点击“任务推进”。平台会生成当前任务对应的任务推进提示词。

### 8. 复制任务推进提示词到项目对话

复制任务推进提示词，回到 Codex 中基于该项目的聊天对话，将提示词粘贴进去并回车。

Codex 收到任务推进提示词后，会通过 MCP 获取完整任务上下文，并按照平台返回的任务说明、需求信息和项目规则开始执行。

### 9. 自动化推进开始

任务推进启动后，Codex 会自动执行开发、验证和状态回写流程。当前任务完成后，会话会继续检查并接取新的任务或缺陷，持续推进后续工作。

自动化推进会持续运行，直到出现以下任一情况：

- 后台停止该 Codex 会话。
- 在当前会话中输入“停止接取新的任务”。
- 平台返回不再继续轮询或继续接取的会话状态。

## 中断恢复

如果任务推进到一半时会话断开、客户端关闭或网络中断，可以重新进入协同平台，在该任务的“任务推进”入口重新复制任务推进提示词。

将新的任务推进提示词再次粘贴到基于同一项目的 Codex 聊天对话中并回车，Codex 会重新读取任务上下文并继续推进。恢复时应优先使用同一个本地项目目录，避免重复拉取或在不同工作区中产生分叉修改。

## 使用建议

- 一个项目建议固定使用一个本地仓库目录，避免多个目录同时推进同一任务。
- 不要把 Agent Token、数据库密码、SSH 私钥等敏感信息提交到 Git 仓库。
- MCP 接入提示词和任务推进提示词应来自协同平台最新生成的内容。
- 如果 Codex 提示 MCP 不可用，先重新执行 MCP 接入提示词，再重新执行任务推进提示词。
- 长时间自动化推进时，保持 Codex 客户端和本机网络稳定。
